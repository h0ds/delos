# Oracle Mission & Principles

**The Oracle:** An AI-powered research agent that monitors, analyzes, and alerts on market-moving situations in real-time.

**Core Mission:** Help traders understand situations faster by deploying an intelligent research agent that continuously monitors multiple sources.

---

## What Oracle Does

Oracle is an AI research agent that:
- **Aggregates** signals from multiple real-time sources
- **Analyzes** sentiment, impact, and market relevance
- **Scores** signals based on confidence and urgency
- **Surfaces** market-moving information instantly
- **Alerts** users to critical situations requiring action

Think of Oracle as your personal AI research assistant, working 24/7 to monitor the market and surface what matters.

## Core Principles

### 1. **Oracle Acts, Doesn't Just Display**

Oracle isn't passive. It's actively:
- Monitoring sources continuously
- Analyzing each signal
- Scoring urgency
- Highlighting critical findings

**User Experience:**
- Users ask Oracle questions
- Oracle responds with analyzed findings
- Intelligence is pre-calculated
- Context is always provided

**Example:**
✅ "Oracle analyzed 47 signals for Bitcoin. Sentiment is bullish. 3 signals are critical."  
❌ "Here are 47 signals. You figure out what matters."

### 2. **Speed is Competitive Advantage**

Oracle exists to make traders faster than competitors.

**In Practice:**
- Real-time signal aggregation (< 1 second)
- Instant sentiment/impact analysis (< 100ms)
- Dashboard render in < 2 seconds
- No analysis paralysis time wasted
- Action-ready information

**Why It Matters:**
In trading, 1 second is the difference between a signal and old news. Oracle compresses research time from minutes to seconds.

### 3. **Context Prevents Mistakes**

Oracle always provides:
- **What** happened (the signal)
- **Why** it matters (impact score)
- **How** it feels (sentiment)
- **Where** it applies (markets)
- **When** it occurred (timestamp)
- **Who** reported it (source credibility)

**Example:**
✅ "Bearish · Critical Impact · Reuters · 2m ago · BTC, ETH, SPX"  
❌ "Fed Raises Rates"

### 4. **Intelligence Over Volume**

Oracle's job is NOT to show everything. It's to show what matters.

**In Practice:**
- 47 signals discovered, top 10 displayed
- Filtering by impact/relevance
- High-confidence alerts prioritized
- Summary assessments provided
- Detailed data available if needed

### 5. **Transparent Analysis**

Oracle shows its work:
- Source verification (Reuters, Bloomberg, Reddit)
- Confidence levels (displayed as metrics)
- Methodology transparency (sentiment algorithm)
- Access to raw data (links provided)
- Assessment reasoning (AI summary)

**Users Know:**
- Why this signal was surfaced
- How confident Oracle is
- What data informed the analysis
- Where to verify independently

### 6. **24/7 Monitoring**

Oracle never sleeps:
- Continuous source monitoring
- Real-time signal processing
- Live alerts for critical situations
- Dashboard updates instantly
- Historical tracking of trends

**User Benefit:**
- Sleep knowing Oracle is watching
- Get alerted if critical signals arrive
- No FOMO about missed events
- Comprehensive data when you check

### 7. **Actionable, Not Analytical**

Oracle isn't a research library. It's a decision support tool.

**Information Hierarchy:**
1. **Immediate** — What needs your attention now (critical impact, fresh signals)
2. **Important** — Significant signals worth understanding (medium impact, relevant)
3. **Contextual** — Background signals for understanding trends (low impact, historical)
4. **Reference** — Raw data for independent analysis (all available)

Users see critical first. Research layer is optional.

### 8. **Professional, Not Sensational**

Oracle uses measured language:
- "Critical" not "URGENT!!!"
- "Bullish" not "MOON INCOMING"
- Data-driven not hype-driven
- Professional appearance
- Trustworthy tone

## Decision Framework

**When building Oracle features, ask:**

1. **Does it help traders act faster?**
   - If NO → Don't build
   - If YES → Continue

2. **Does it add intelligence or just data?**
   - If JUST DATA → Don't build
   - If INTELLIGENCE → Continue

3. **Is it obviously correct?**
   - If AMBIGUOUS → Skip
   - If CLEAR → Continue

4. **Can it be wrong?**
   - If OFTEN WRONG → Don't build
   - If USUALLY RIGHT → Build

5. **Does it require account/tracking?**
   - If YES → Redesign or skip
   - If NO → Build

**Example: Should we add email alerts?**
1. Does it help traders act faster? YES → Continue
2. Does it add intelligence? YES (curated alerts) → Continue
3. Is it obviously correct? YES (user-defined) → Continue
4. Can it be wrong? NO (user controls) → Continue
5. Require tracking? NO (optional feature) → **Build it**
